## 1. Definition

The **Unicorn Model** is the canonical ICT composite setup combining two of the highest-priority PD Arrays:

> A **Breaker Block** overlapped with a **Fair Value Gap** of the same direction, ideally located inside the OTE zone of the active dealing range.

It is the strongest standalone composition the bot will trade in V1.

## 2. Notation

| Symbol | Meaning |
| ------ | ------- |
| `B`       | Breaker Block from concept 06 |
| `G`       | FVG (BISI or SIBI) from concept 04, **same direction as B and generated by the same displacement leg that created B's invalidation event** |
| `Z`       | OTE zone of the displacement leg (concept 11) |
| `U`       | Unicorn setup object: `{kind, range, entry_price, sl, tp, leg_ref, components: (B, G), inside_ote: bool}` |

## 3. Formal definition

A Unicorn `U` of direction `d ∈ {BULL, BEAR}` is registered when **all** of:

```
1. B ∈ ACTIVE_t                          (Breaker valid at bar t — concept 06)
2. G ∈ ACTIVE_t                          (FVG valid at bar t — concept 04)
3. B.kind == d   AND   G.kind matches d
       (BISI matches BULL, SIBI matches BEAR)
4. SAME_LEG(B, G) == true
       B was created by the invalidation of an OB caused by leg L*;
       G must have been generated by that same L* (or by a leg overlapping in time/price).
5. intersect(B.range, G.range)  ≠  ∅
       U.range = intersect(B.range, G.range)
6. (optional, on by default) U.range overlaps the OTE zone Z of L*:
       intersect(U.range, Z)  ≠  ∅      ⇒  U.inside_ote = true
7. New-entry-gating passes (concept 13):
       new_entries_allowed(t) == true
8. Bias filter passes (concept 13):
       d == BULL ⇒ current_price < mid_open(date(t))
       d == BEAR ⇒ current_price > mid_open(date(t))
9. HTF anchor present (concept 09):
       U.range intersects at least one HTF PD Array on the configured htf_ref TFs.
```

### 3.1 Entry, SL, TP

```
Entry price (limit order at the retest):
    d == BULL: U.entry_price = max(B.range.low, G.range.low)         # proximal edge of intersection
    d == BEAR: U.entry_price = min(B.range.high, G.range.high)

Stop Loss (between 10 and 20 pips/ticks behind the FVG-creating bar's extreme, per research markdown):
    Let f = the bar that anchored G (i.e., bar t such that G = FVG(t)).
    d == BULL: U.sl = bars[f].low  - sl_offset_ticks * tick_size
    d == BEAR: U.sl = bars[f].high + sl_offset_ticks * tick_size

Take Profit (next opposite liquidity pool or next-rank HTF PD Array):
    target_pool = nearest active opposite-side pool from concept 10
                  beyond U.entry_price in direction d
    U.tp = target_pool.price
    (Fallback if no pool within K * ATR: nearest HTF PD Array of rank ≤ 3.)
```

### 3.2 R-multiple discipline

```
R_risk    = |U.entry_price - U.sl|
R_reward  = |U.tp          - U.entry_price|
U.RR      = R_reward / R_risk
```

V1 rejects setups with `U.RR < min_rr` (config, default 1.5).

## 4. Detection (pseudocode)

```text
input:
    breakers_active, fvgs_active           # registries
    legs_with_metadata                      # concept 03, plus link to which Breaker each leg's invalidation produced
    pd_array_selector                       # concept 09
    sessions                                # concept 13
    mid_open_today
    config

unicorns = []
for B in breakers_active:
    L = B.origin_ob.leg_ref                # the leg that broke OB → Breaker
    for G in fvgs_active:
        if G.kind not matches B.kind:
            continue
        if not same_leg(B, G, L):           # G must come from the displacement leg L (or an overlap)
            continue
        intersect_range = intersection(B.range, G.range)
        if intersect_range is None:
            continue

        # Check OTE confluence
        Z = ote_zone(L)
        inside_ote = intersect_range overlaps Z

        # Check HTF anchor
        if not htf_anchor_exists(intersect_range, htf_ref_tfs):
            continue

        # Time gating happens at *entry decision* time, not detection time;
        # we register all valid compositions and let the trigger logic check sessions
        U = UnicornSetup(
            kind=B.kind, range=intersect_range,
            entry_price=proximal_edge(intersect_range, B.kind),
            sl=sl_from_fvg_anchor(G, B.kind, config.sl_offset_ticks),
            tp=tp_target(B.kind, intersect_range, config),
            components=(B, G),
            inside_ote=inside_ote,
            leg_ref=L,
        )
        if U.RR < config.min_rr:
            continue
        unicorns.append(U)

# At every bar t with new_entries_allowed(t):
for U in active_unicorns:
    if bias_filter_passes(U.kind, current_price(t), mid_open_today):
        if price_touched_entry(U, t):
            emit_trade_signal(U)
```

## 5. Invalidation

A Unicorn is invalidated if **any** of:

- `B` invalidates (concept 06 §5).
- `G` invalidates (concept 04 §5).
- A body close pierces `U.range` on the opposing side (treat the intersection itself as a level — the strongest invalidation).
- Time-stop: if the price does not touch `U.entry_price` within `max_age_bars` after registration, the Unicorn is pruned (default 50 bars on the reference TF; configurable).
- Session change: if a new trading day begins and the underlying leg `L` was from the prior day, the Unicorn is downgraded (kept for observability, not traded).

## 6. Confluence rules

- `inside_ote == true` → preferred; `inside_ote == false` → still tradable if other confluences are strong.
- Sweet-spot of OTE (0.705) inside `U.range` → highest priority.
- Unicorn during PO3 distribution (concept 12) in the bias direction → maximum confidence.
- Unicorn during Silver Bullet window (concept 13) → `+1` confluence.
- L2 confirmation (Phase 8): on the entry bar, `delta_acceleration` aligned with `U.kind` and `spread_compression` elevated → final go-ahead.

## 7. Parameters (configs/default.yaml)

```yaml
setups:
  unicorn:
    require_inside_ote: false       # true → only OTE-aligned Unicorns
    sl_offset_ticks: 8              # 10–20 "pips" — for NQ at 0.25 pt/tick that's 2–5 pts
    min_rr: 1.5
    max_age_bars: 50                # on reference TF
    tp_strategy: nearest_opposite_pool   # "nearest_opposite_pool" | "htf_pd_array" | "fixed_R"
    fallback_tp_R: 3.0              # used when tp_strategy fails to find a target
```

## 9. Open questions

- **(Q14.a)** "Same leg" — strict (G must be the same leg that broke the OB) or relaxed (G overlaps in time within a window)? **Default v1:** relaxed within `same_leg_window_bars` (= leg length × 1.5). Strict mode available as config.
- **(Q14.b)** TP target = nearest opposite pool OR HTF PD Array, whichever is closer? **Default v1:** `nearest_opposite_pool` first; if none within `K × ATR`, fall back to HTF PD Array of rank ≤ 3.
- **(Q14.c)** SL offset — research markdown says "10–20 pips". For NQ futures, "pip" is non-standard; v1 interprets as ticks. 8 ticks ≈ 2 NQ points, a sensible default. Should we widen or tighten for NQ specifically? **Default v1:** 8 ticks (= 2 NQ points = $40/contract risk per tick is wrong — it is `8 × $5 = $40` distance per contract).
